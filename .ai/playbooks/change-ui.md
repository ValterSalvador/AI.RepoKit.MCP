# Change UI

1. Identify the smallest affected view or component.
2. Use `get_context symbols brief` to find related components, services, handlers, and models before opening files.
3. Use `get_context endpoints brief` when the UI change touches API calls.
4. Check existing design patterns before editing.
5. Preserve accessibility, responsive behavior, and validation states.
6. Run focused build or tests when available.
7. Summarize files changed and verification.
